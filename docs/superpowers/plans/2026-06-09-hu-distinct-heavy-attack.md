# Distinct Hu Heavy Attack (presenter-rendered) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give Hu's heavy attack its own art and presenter-rendered animation + collision, instead of sharing the light attack frames and rendering through the FighterVisual fallback.

**Architecture:** Generate a distinct `attack-heavy` action through the existing smooth-master → scale → pixelize → canonical-slot pipeline into new `heavy_*` slots. Add `heavy_*` poses to the manifest, an `ATTACKING_HEAVY` state to the animation graph, and a `hu_attack_heavy` timeline clip. Route `ATTACKING_HEAVY` to the presenter, and make `PresentationCollision` pick the **heavy** strike pose for `hu_heavy` (it currently hardcodes the light strike pose). Re-derive `hu_heavy` reach from the new geometry.

**Tech Stack:** Godot 4.6.2 (GDScript), `aiexp sprite-extractor` (smooth + pixelize), JSON data, headless runner (`./run.sh --test`).

**Prerequisite:** the smooth-master pipeline (`tools/scale_masters.gd`, `tools/install_pixelized.gd`, `--measure-anchors`/`--anchor-sanity`) from `docs/superpowers/plans/2026-06-09-smooth-master-pixelize-pipeline.md`, already implemented for Hu's other actions.

**Scope:** Hu heavy only. Reuses the existing pipeline + presenter; no changes to the light path beyond making the strike-pose lookup per-attack.

## Task 4: Per-attack strike pose in PresentationCollision (TDD)

The capsule for `hu_heavy` must use the **heavy** strike pose, not the light one. Today `attack_capsule_world` hardcodes `STRIKE_POSE = "strike_extended"` (`presentation_collision.gd:8,49`).

**Files:**
- Modify: `WUGodot/scripts/visual/presentation_collision.gd`
- Test: `WUGodot/tests/test_heavy_capsule_pose.gd`

- [ ] **Step 1: Write the failing test + register it**

Create `WUGodot/tests/test_heavy_capsule_pose.gd` (assumes the manifest from Task 1 has both `strike_extended` and `heavy_strike` with different `weaponTip`). The test asserts the heavy capsule's far endpoint matches the **heavy_strike** pose's world tip — not just that the reach differs (the heavy radius bonus in `hitbox_template.gd` would make reach differ even with the light pose, a false positive).

```gdscript
extends RefCounted

const PC = preload("res://scripts/visual/presentation_collision.gd")
const FighterScript = preload("res://scripts/fighter.gd")
const AC = preload("res://scripts/attack_catalog.gd")

func _mid_active(atk: Variant) -> Variant:
	var f: Variant = FighterScript.new()
	f.position = Vector2(400, 900); f.facing = 1
	f._attack_state.start(atk)
	f._attack_state.advance(atk.windup_end + 0.01)
	return f

# Mirror PresentationCollision._to_world (facing 1 -> no mirror): the capsule's "b" endpoint
# is the pose weaponTip mapped to world from the pose foot anchor.
func _world_tip(pose: Dictionary, root: Vector2, scale: float) -> Vector2:
	var foot: Vector2 = pose.get("footAnchor", Vector2.ZERO)
	var tip: Vector2 = pose.get("weaponTip", Vector2.ZERO)
	return root + (tip - foot) * scale

func run_all() -> Dictionary:
	var passed := 0
	var failed := 0
	var failures: Array[String] = []

	var pc: Variant = PC.new()
	pc.register_from_manifest_file("hu", "res://assets/animation_manifests/hu.manifest.json")
	var m: Variant = pc._manifests["hu"]
	var scale: float = m.render_scale
	var heavy_pose: Dictionary = m.get_pose("heavy_strike")
	var light_pose: Dictionary = m.get_pose("strike_extended")

	var heavy := _mid_active(AC.hu_heavy())
	pc.register_fighter(heavy, "hu")
	var tip_b: Vector2 = (pc.attack_capsule_world(heavy)["b"]) as Vector2

	# The heavy capsule endpoint must equal the heavy_strike world tip.
	if tip_b.distance_to(_world_tip(heavy_pose, heavy.position, scale)) <= 1.0:
		passed += 1
	else:
		failed += 1; failures.append("heavy capsule endpoint should match heavy_strike tip (got %s)" % str(tip_b))

	# ...and must NOT match the light strike tip (unless the two tips are identical in art).
	if not (heavy_pose.get("weaponTip", Vector2.ZERO) as Vector2).is_equal_approx(light_pose.get("weaponTip", Vector2.ZERO) as Vector2):
		if tip_b.distance_to(_world_tip(light_pose, heavy.position, scale)) > 1.0:
			passed += 1
		else:
			failed += 1; failures.append("heavy capsule must not use the light strike pose")
	else:
		passed += 1  # identical tips in art -> indistinguishable; skip

	return {"passed": passed, "failed": failed, "failures": failures}
```

Register it **now** (before the expected-fail run) — add to `_TEST_MODULES` in `WUGodot/tests/run_tests.gd`:

```gdscript
	"res://tests/test_heavy_capsule_pose.gd",
```

- [ ] **Step 2: Run to verify it fails**

Run: `./run.sh --test`. Expected: FAIL — heavy still uses the hardcoded light pose, so the endpoint matches `strike_extended`, not `heavy_strike`.

- [ ] **Step 3: Implement per-attack strike pose**

In `presentation_collision.gd`, add the map and a resolver, and use it in `attack_capsule_world`:

```gdscript
const STRIKE_POSE: String = "strike_extended"   # default / fallback
const STRIKE_POSE_BY_ID: Dictionary = {
	"hu_light": "strike_extended",
	"hu_heavy": "heavy_strike",
}

func _strike_pose_for(fighter: Variant) -> String:
	var def: Variant = fighter._attack_state.def
	if def != null and STRIKE_POSE_BY_ID.has(str(def.id)):
		return STRIKE_POSE_BY_ID[str(def.id)]
	return STRIKE_POSE
```

In `attack_capsule_world`, replace `var pose: Dictionary = manifest.get_pose(STRIKE_POSE)` with:

```gdscript
	var pose: Dictionary = manifest.get_pose(_strike_pose_for(fighter))
```

(Leave `_defender_hurtbox` as-is — it falls back to `guard`/body-rect for the un-manifested enemy.)

- [ ] **Step 4: Run to verify it passes**

Run: `./run.sh --test`. Expected: PASS — `rl != rh`.
